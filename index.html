What is a Convolutional Neural Network?

In machine learning, a classifier is a function that assigns a class label to an input. Formally, we can think of it as a function \( f \) that maps an input vector \( x \) to one of \( K \) classes: \( f : \mathbb{R}^n \to \{1, 2, \dots, K\} \).

A Convolutional Neural Network (CNN) is a special type of neural network that is very good at classifying image-like data. For an RGB image with height \(H\), width \(W\), and 3 color channels, we can represent the input as a 3D tensor: \[ x \in \mathbb{R}^{H \times W \times 3}. \] The CNN learns a function \[ f_\theta : \mathbb{R}^{H \times W \times 3} \to \mathbb{R}^K, \] where \(K\) is the number of classes and \(\theta\) denotes all learnable parameters (weights and biases).

Basic Building Blocks

Tensors

A tensor is a generalization of vectors and matrices to multiple dimensions. In our CNN:

  • The input image is a 3D tensor \( x \in \mathbb{R}^{H \times W \times C} \).
  • Each hidden layer produces an activation tensor \( a^{(l)} \in \mathbb{R}^{H_l \times W_l \times C_l} \).
  • The final layer produces a 1D output vector \( z \in \mathbb{R}^K \) of class scores.

Neurons

A neuron computes a weighted sum of its inputs plus a bias, then applies a non-linear activation function. In its simplest form: \[ y = \phi(w^\top x + b), \] where \(x\) is the input vector, \(w\) is the weight vector, \(b\) is the bias, and \(\phi\) is a non-linear function such as ReLU.

Layers

A layer is a collection of neurons that apply the same operation (same type of computation, same hyperparameters) to different positions or units. In CNNs we mainly use:

  • Convolutional layers
  • Non-linear activation layers (e.g., ReLU)
  • Pooling layers
  • Fully connected (dense) layers
  • Softmax output layer

Convolutional Layers (Core of a CNN)

The convolutional layer learns a set of kernels (filters) that detect local patterns in the image, such as edges, corners, or textures. Suppose the input to the layer is a tensor \[ a^{(l-1)} \in \mathbb{R}^{H_{l-1} \times W_{l-1} \times C_{l-1}}, \] and we learn \(C_l\) kernels, each of size \[ K_h \times K_w \times C_{l-1}. \] The \(j\)-th kernel is: \[ W^{(l)}_j \in \mathbb{R}^{K_h \times K_w \times C_{l-1}}, \quad b^{(l)}_j \in \mathbb{R}. \]

The output activation at spatial position \((u, v)\) in channel \(j\) is: \[ z^{(l)}_{u,v,j} = \sum_{c=1}^{C_{l-1}} \sum_{p=1}^{K_h} \sum_{q=1}^{K_w} W^{(l)}_{j,p,q,c} \cdot a^{(l-1)}_{u+p-1,\, v+q-1,\, c} + b^{(l)}_j. \] Then a non-linearity is applied: \[ a^{(l)}_{u,v,j} = \phi\big(z^{(l)}_{u,v,j}\big). \]

Hyperparameters such as kernel size \((K_h, K_w)\), stride \(S\) (how far we move the kernel each step), and padding \(P\) (how many pixels we add around the border) control the spatial size of the output feature maps.

Activation Function (Non-Linearity)

To allow the network to learn complex, non-linear functions, we apply an activation function after each convolution or fully connected layer. A common choice is the ReLU (Rectified Linear Unit): \[ \phi(x) = \max(0, x). \]

Applied elementwise to each activation: \[ a^{(l)}_{u,v,j} = \max\big(0, z^{(l)}_{u,v,j}\big). \]

Pooling Layers

Pooling layers downsample the spatial dimensions of the activation maps, making the representation more compact and robust to small translations in the input. For example, max pooling with a window size \(P \times P\) and stride \(S\) is defined as: \[ a^{(l)}_{u,v,j} = \max_{\substack{0 \le p < P \\ 0 \le q < P}} a^{(l-1)}_{u+p,\, v+q,\, j}. \]

This operation keeps the largest response in each small region and discards the rest, reducing \(H\) and \(W\) while keeping the number of channels \(C\) the same.

Fully Connected Layers

After several rounds of convolution, activation, and pooling, we obtain a high-level feature representation. We then flatten the final activation tensor to a vector and feed it into one or more fully connected (dense) layers.

If the flattened feature vector is \[ h \in \mathbb{R}^M, \] and the fully connected layer has \(K\) outputs, then: \[ z = W h + b, \quad W \in \mathbb{R}^{K \times M}, \quad b \in \mathbb{R}^K. \] Each component \(z_k\) is the score for class \(k\).

Softmax Output and Class Probabilities

To convert raw scores \(z \in \mathbb{R}^K\) into a probability distribution over classes, we apply the softmax function: \[ \hat{y}_k = \frac{\exp(z_k)}{\sum_{j=1}^K \exp(z_j)}. \]

Here \( \hat{y}_k \) is interpreted as the estimated probability that the input image belongs to class \(k\). The predicted class is: \[ \hat{c} = \arg\max_k \hat{y}_k. \]

How Does a CNN Learn?

All kernels and biases in the network are collected into a parameter set \(\theta\). During training, we are given input–label pairs \((x^{(i)}, y^{(i)})\), where \(y^{(i)}\) is the true class for image \(x^{(i)}\).

We define a loss function, typically the cross-entropy loss: \[ \mathcal{L}(\theta) = - \sum_{i} \sum_{k=1}^K \mathbf{1}\{y^{(i)} = k\} \log \hat{y}^{(i)}_k, \] where \(\hat{y}^{(i)}_k\) is the softmax probability output by the network for class \(k\) on example \(i\).

Using backpropagation and an optimization algorithm such as gradient descent, we iteratively update the parameters: \[ \theta \leftarrow \theta - \eta \nabla_\theta \mathcal{L}(\theta), \] where \(\eta\) is the learning rate. Over time, the CNN learns kernels and weights that minimize the loss and improve classification accuracy.

Summary

A Convolutional Neural Network is a differentiable function \( f_\theta \) that maps input images to class probabilities through a sequence of tensor operations: convolutions, non-linear activations, pooling, and fully connected layers, ending with a softmax. By learning the parameters \(\theta\) from data, CNNs automatically discover useful visual features for tasks such as image classification, segmentation, and object detection.
